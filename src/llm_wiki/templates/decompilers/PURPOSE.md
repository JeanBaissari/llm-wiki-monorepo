# Decompilers — Domain Purpose

This template governs wikis for reverse-engineering, decompilation, and binary analysis. It is designed for security researchers, malware analysts, compiler engineers, and CTF players who need a structured knowledge base of binary formats, opcodes, tools, and research findings.

## What this domain covers

- **Binary format specifications** — PE, ELF, Mach-O, custom firmware formats
- **Instruction set opcodes** — mnemonics, encodings, operands, stack effects
- **Decompiler tools** — IDA Pro, Ghidra, Binary Ninja, radare2, custom tools
- **Research findings** — decompilation accuracy, pattern discoveries, algorithm improvements

## When to use this template

Use when your wiki focuses on understanding, documenting, or improving decompilation tooling and binary analysis workflows.

## Relationship to base types

The base entity/concept/source types still apply:
- **Entities** — tool authors, companies (Hex-Rays, NSA), processor vendors
- **Concepts** — SSA form, control-flow graphs, type recovery, lifting
- **Sources** — academic papers, tool documentation, conference talks
