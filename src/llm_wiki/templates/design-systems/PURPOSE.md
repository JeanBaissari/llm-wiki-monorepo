# Domain: Design Systems

The design-systems domain documents UI design tokens, components, patterns, and guidelines. It serves as a single source of truth for product design language — enabling consistency, faster development, and shared vocabulary across engineering and product teams.

## When to use this domain

- Cataloguing design tokens (colors, typography, spacing, shadows)
- Documenting component APIs, states, and usage guidelines
- Recording interaction and layout patterns
- Maintaining accessibility and design review guidelines
- Tracking component maturity from draft through stable
