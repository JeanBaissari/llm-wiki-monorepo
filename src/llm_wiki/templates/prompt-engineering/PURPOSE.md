# Domain: Prompt Engineering

The prompt-engineering domain captures structured knowledge about prompting techniques, evaluation methodology, reusable prompt templates, and LLM provider capabilities. It supports systematic comparison of prompting strategies and serves as a reference for practitioners building reliable LLM-based systems.

## When to use this domain

- Cataloguing prompting techniques (e.g., chain-of-thought, few-shot, tree-of-thought)
- Documenting evaluation benchmarks and results
- Storing reusable, versioned prompt templates
- Tracking LLM provider APIs, pricing, and capabilities
- Comparing prompting strategies across models and tasks
