# Domain: Architecture

The architecture domain captures system design, service topology, infrastructure configuration, and architectural decision records (ADRs). It serves as the authoritative reference for how systems are built, how they interact, and what trade-offs have been made.

## When to use this domain

- Documenting microservice boundaries, APIs, and dependencies
- Recording infrastructure topology and provisioning
- Capturing architecture decisions with context and consequences
- Maintaining service-level ownership, SLAs, and runbooks
- Diagramming system interactions and data flows
