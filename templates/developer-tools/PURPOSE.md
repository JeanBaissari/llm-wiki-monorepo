# Developer Tools — Domain Purpose

This template governs wikis for developer tooling, workflow automation, and platform integrations. It is designed for software engineers, DevOps practitioners, platform teams, and tool builders who need a structured knowledge base of tools, practices, performance data, and interoperability.

## What this domain covers

- **Developer tools** — editors, linters, CI/CD platforms, debuggers, profilers, package managers
- **Workflows** — setup guides, best practices, troubleshooting patterns
- **Benchmarks** — performance comparisons, scalability tests, feature matrices
- **Integrations** — how tools connect, plugin ecosystems, migration guides

## When to use this template

Use when your wiki's primary focus is documenting, comparing, and optimizing the tooling ecosystem used in software development workflows.

## Relationship to base types

The base entity/concept/source types still apply:
- **Entities** — tool authors, organizations (JetBrains, GitHub, HashiCorp), plugins
- **Concepts** — continuous integration, static analysis, monorepo management
- **Sources** — documentation, blog posts, conference talks, RFCs
